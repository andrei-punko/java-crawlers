# tabor.ru-crawler

Модуль собирает женские анкеты с [tabor.ru](https://tabor.ru) (Минск), скачивает заглавное фото и сохраняет результат в JSON. Рядом есть локальный дашборд для просмотра и ручной разметки (скрыть / избранное).

## Состав

| Часть | Назначение |
|---|---|
| `MainApp` | Точка входа краулера: resume, фильтры, фото, запись JSON |
| `TaborRuWebCrawler` | Обход поиска и разбор анкет (наследник `crawler-engine`) |
| `ProfileData` | DTO профиля |
| `PhotoDownloader` | Скачивание cover-фото в `photos/{id}.ext` |
| `TaborDashboardServer` | Локальный HTTP-сервер + UI из `resources/dashboard/` |

Артефакт сборки: fat-jar  
`target/tabor.ru-crawler-1.0-SNAPSHOT-jar-with-dependencies.jar`.

## Как устроен поиск

1. Стартовый URL — поиск женщин в Минске с диапазоном возраста (`minAge`–`maxAge`).
2. На карточке списка сразу отсекаются: не Минск, возраст вне диапазона (если указан), нет cover-фото `/photos/`, уже виденные id.
3. Для оставшихся открывается страница анкеты → заполняется `ProfileData`.
4. После обхода `MainApp` применяет post-фильтры (`rejectionReason`), для прошедших качает фото и пишет JSON.

Пагинация: если сайт перестаёт отдавать `rel=next`, краулер сам наращивает `page=`.  
При `pagesCap=-1` действует hard-cap `2000` страниц.

## Resume

`profiles.json` и `photos/` **не удаляются** при запуске.

- существующие id из JSON попадают в seed и не запрашиваются повторно;
- фото не качается, если файл уже есть;
- новые подходящие анкеты дописываются в JSON (сортировка по возрасту, затем имени).

Если `profiles.json` есть, но не читается — процесс **падает** (чтобы не затереть файл).

## Фильтры после разбора анкеты

Отсекаются (если поле заполнено и совпадает):

| Поле | Условие |
|---|---|
| город / фото | нет города или нет cover `/photos/` |
| город | не Минск |
| возраст | вне `minAge`–`maxAge` (пустой возраст не режет) |
| вес | **> 80 кг** |
| семейное положение | `замужем` |
| образование | `среднее` |
| дети | `есть, живем вместе`, `есть, живем порознь`, `нет, и не хочу` |
| курение | `курю` |
| алкоголь | `люблю выпить` |
| мат. поддержка | `ищу спонсора` |

Фото качаются **только** для прошедших фильтр.

## Файлы рядом с запуском

| Файл / папка | Содержание                                                 |
|---|------------------------------------------------------------|
| `profiles.json` | Список профилей                                            |
| `photos/` | Заглавные фото `{id}.jpg` (и др.)                          |
| `hidden.json` | Скрытые id (дашборд)                                       |
| `favorites.json` | Избранные id (дашборд)                                     |
| `tabor-crawler.log` | Лог краулера (не удаляется между запусками, накопительный) |

`profiles.json`, `photos/`, `hidden.json`, `favorites.json` в git не коммитятся.

## Сборка и запуск

Из корня репозитория:

```bat
mvn -pl tabor.ru-crawler -am package
```

### Краулер

```bat
cd tabor.ru-crawler
run-search.bat
```

По сути:

```bat
java -jar target/tabor.ru-crawler-1.0-SNAPSHOT-jar-with-dependencies.jar profiles.json -1 250 30 45
```

Аргументы: `outputJson [pagesCap] [delayMs] [minAge] [maxAge]`  
`pagesCap=-1` — «все страницы» (с hard-cap).

### Дашборд

```bat
cd tabor.ru-crawler
run-dashboard.bat
```

Откроется http://localhost:8080  

Остановка: Ctrl+C в консоли дашборда или `stop-dashboard.bat`.  
Пока дашборд запущен, fat-jar занят — `mvn clean` может не удалить `target/…jar`.

Возможности UI:

- карточки с фото и полями (имя/возраст — в заголовке под фото);
- клик по фото → открывается анкета в новой вкладке;
- «Скрыть» / «В избранное»;
- режим «Избранное» сверху;
- чекбокс «Показать „разведена“» (не показаны по умолчанию);
- Ctrl+Z — отмена последнего скрытия (можно применять многократно).

Избранные живут на отдельном борде: с обычного списка уходят при добавлении в избранное.
